Database Models

Types of Database Models

A database model is a conceptual representation of the structure and organization of a database.

  • Hierarchical Model: A hierarchical model is a database model that organizes data in a tree-like structure, with each record having a single parent record.
  • Network Model: A network model is a database model that allows each record to have multiple parent and child records, forming a network of relationships.
  • Relational Model: A relational model is a database model that organizes data into tables with well-defined relationships between them.

Relational Model

A relational model is a database model that organizes data into tables with well-defined relationships between them.

Types of Relationships

  • One-to-One (1:1): A one-to-one relationship is a relationship between two tables where each record in one table is related to only one record in the other table.
  • One-to-Many (1:N): A one-to-many relationship is a relationship between two tables where each record in one table is related to multiple records in the other table.
  • Many-to-Many (M:N): A many-to-many relationship is a relationship between two tables where each record in one table is related to multiple records in the other table, and vice versa.

Example of Relational Model

Consider a database that stores information about customers and orders. The relational model would organize this data into two tables: Customers and Orders.
