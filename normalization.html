Normalization in DBMS

First Normal Form (1NF)

A table is in 1NF if it does not contain any repeating groups or arrays.

Student ID Name Subjects
101 John Math, Science, English
102 Jane Math, History, Geography

Problem: Repeating group "Subjects" is not allowed in 1NF.

First Normal Form (1NF) Solution

Student ID Name Subject
101 John Math
101 John Science
101 John English
102 Jane Math
102 Jane History
102 Jane Geography

Second Normal Form (2NF)

A table is in 2NF if it is in 1NF and all non-prime attributes are fully functional dependent on the primary key.

Order ID Customer ID Order Date Total Amount
101 1 2022-01-01 1000
102 1 2022-01-15 500
103 2 2022-02-01 2000

Problem: Non-prime attribute "Total Amount" is not fully functional dependent on the primary key "Order ID".

Second Normal Form (2NF) Solution

Order ID Customer ID Order Date
101 1 2022-01-01
102 1 2022-01-15
103 2 2022-02-01
Order ID Total Amount
101 1000
102 500
103 2000

Third Normal Form (3NF)

A table is in 3NF if it is in 2NF and there are no transitive dependencies.

Customer ID Customer Name Address
1 John 123 Main St
2 Jane 456 Elm St

Problem: Transitive dependency between "Customer ID" and "Address" through "Customer Name".

Third Normal Form (3NF) Solution

Customer ID Customer Name
1 John
2 Jane
Customer ID Address
1 123 Main St
2 456 Elm St
