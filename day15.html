Data Security and Integrity

Data Security Types

Data security refers to the protection of digital data from unauthorized access, use, disclosure, disruption, modification, or destruction.

Type Description
Confidentiality Protecting sensitive information from unauthorized access.
Integrity Ensuring that data is accurate, complete, and not modified without authorization.
Availability Ensuring that data is accessible and usable when needed.
Authentication Verifying the identity of users, systems, or entities.
Authorization Controlling access to resources based on user identity, role, or permissions.

Data Integrity

Data integrity refers to the accuracy, completeness, and consistency of data.

Type Description
Entity Integrity Ensuring that each entity (e.g., customer, order) has a unique identifier.
Referential Integrity Ensuring that relationships between entities are consistent and valid.
Domain Integrity Ensuring that data values are within a specified range or format.

Anomalies

Anomalies refer to inconsistencies or irregularities in data that can compromise data integrity.

Type Description
Insertion Anomaly Inserting a new record with incomplete or inconsistent data.
Update Anomaly Updating a record with inconsistent or outdated data.
Deletion Anomaly Deleting a record with incomplete or inconsistent data.
